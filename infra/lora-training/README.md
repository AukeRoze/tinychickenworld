# Cast LoRA training

> ⚠️ **Read `TRAINING-PLAN.md` first.** The previous LoRA produced generic
> chicks because the dataset, captions AND hero anchors were off-model (wrong
> colours/accessories, watercolor style, even swapped). The clean flow:
> 1. Put **on-model** hero anchors in `dataset-refs/` (`pip-hero.png`,
>    `mo-hero.png`, `bo-hero.png`) that EXACTLY match `bible/channel.yml`
>    (Pip = cream-white + straw hat + red bandana; Mo = blue-grey + red scarf;
>    Bo = tan + round glasses + green scarf; no clothing). Verify each one by eye.
> 2. Run `python generate-refs.py` (Gemini hero-reference mode — it writes
>    correct captions itself). Curate ruthlessly.
> 3. Train **per-character** LoRAs (pip/mo/bo) for clean, distinct identities.
>
> The old `dataset/` + `dataset-refs/` were archived to `*-archive-wrong*`.

One-time setup: train the cast so the chicks appear consistently across every
video. This folder holds the playbook + helper scripts.

## What you'll produce

A `cast.safetensors` LoRA file hosted at a public URL. Drop that URL into
`bible/channel.yml` → `imageGen.replicate.castLoraUrl`. Set
`IMAGE_PROVIDER=replicate` in `.env`. From then on, every scene image is
generated by Flux + your LoRA — sub-1¢ per image, near-perfect
character consistency.

## Steps

### 1. Generate reference images for each character (~1 hour)

Run the helper script — uses your existing OpenAI key to bulk-generate
20-30 varied images per character (different poses, expressions,
locations, distances). You'll manually curate to keep ~15-20 of the best
per character.

```bash
cd infra/lora-training
./generate-reference-images.sh
# Output: ./dataset/pip/*.png, ./dataset/mo/*.png, ./dataset/bo/*.png
```

Open the folders and DELETE images where:
- The character looks wrong (wrong color, missing accessory)
- Multiple characters bleed into one
- Composition is bad (cropped face, blurry, etc.)

Keep only your best 15-20 per character. Quality > quantity.

### 2. Caption the images (~30 minutes)

Each PNG needs a sibling .txt file with a caption that includes the
trigger word. Captions teach the LoRA which token activates which
character.

```bash
./caption-images.sh   # auto-generates basic captions; edit as needed
```

Each caption should follow the pattern (banner-true cast — Pip = cream-white +
straw hat, Mo = blue-grey + red scarf, Bo = tan + glasses + green scarf):
```
pip_chicken, a fluffy cream-white baby chick with a straw farmer hat and a red bandana, in the sunflower garden, full body shot, glossy 3D Pixar Illumination cartoon style
```

### 3. Zip the dataset

```bash
./zip-dataset.sh
# Output: ./dataset.zip
```

### 4. Upload + train on Replicate (no local GPU needed)

Replicate runs the training on their GPUs. Cost: ~$2-8 for 1000 steps.

```bash
# Host the zip somewhere public (S3, Cloudflare R2, transfer.sh, etc.)
# Then:
./replicate-train.sh https://your.cdn/dataset.zip
```

The script POSTs to Replicate's training endpoint, polls until done,
and prints the resulting model URL. Training takes 20-40 minutes.

Output looks like:
```
yourusername/tiny-chicken-world-cast:abc123def456...
```

### 5. Drop the URL into the bible

Set in `.env`:
```
IMAGE_PROVIDER=replicate
REPLICATE_API_TOKEN=r8_...
REPLICATE_FLUX_MODEL=lucataco/flux-dev-multi-lora:VERSION
CAST_LORA_URL=https://replicate.delivery/.../cast.safetensors
```

Restart `image-service`. Done.

### 6. Iterate

Generate one test video. If a character drifts, look at the LoRA's
training images — usually the fix is more / better references for that
character. Re-train. The base bible doesn't change; only the LoRA file
URL.

## Cost summary

| Item | Approx. cost |
|---|---|
| Reference image generation (60-90 images at $0.04 ea) | $3-4 |
| Replicate training (1000 steps on A100) | $2-8 |
| **Total one-time** | **$5-12** |
| Per-image inference on Flux+LoRA (vs $0.04-0.08 OpenAI) | $0.003-0.01 |

Break-even vs OpenAI happens around video #50.

## Alternative: local training

If you have a 24GB+ GPU you can train locally with
[ai-toolkit](https://github.com/ostris/ai-toolkit) instead of Replicate.
See `train-locally.yaml` for a config template. Same dataset shape,
trains in ~1-2 hours.
